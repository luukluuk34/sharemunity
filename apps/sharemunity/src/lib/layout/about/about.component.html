Heb je dit probleem wel eens gehad? je leent iets uit aan vrienden en na een paar maanden ben je vergeten dat je het uitgeleend hebt?

Of je hebt iets nodig maar vind het zonde om het te kopen, hier komt sharemunity in het plaatje.

In sharemunity kunnen mensen communities aanmaken of joinen, in deze communities zitten producten (items op de website) die mensen kunnen bekijken en als ze die willen lenen kunnen ze een reservering aanvragen.

Vervolgens kan de eigenaar van het product deze reservering accepteren of weigeren. de reserveringen komen te staan in een overzicht op zowel het profiel als de Reservations pagina

Als je spullen hebt die je niet erg vind om uit te lenen kan je ook een product (item) aanmaken en toevoegen aan een community waaraan je deel neemt zodat de mensen in je community deze spullen eventueel kunnen lenen.

Sharemunity MoSCoW functional requirements

ID Functional Requirement Acceptatie Criteria Prioriteit
F-01 Als user wil ik kunnen registreren zodat ik deel kan nemen aan de website • In de navigatie balk is een knop registreren als een user niet ingelogd is.
• Deze knop brengt je naar het registratie form.
• In het registratie form zit een username input
• In het registratie form zit een email input
• In het registratie form zit een password input
• In het registratie form address input
• De user kan hier registreren.		 Must
F-02 Als user wil ik kunnen inloggen zodra ik een account heb aangemaakt • In de navigatie balk zit een knop inloggen als een user niet ingelogd is.
• Deze knop brengt je naar het inloggen form.
• In de inlog form zit een email input
• In de inlog form zit een password input
• De user kan hier inloggen		 Must
F-03 Als user zou ik snel willen uitloggen als ik niet meer aangemeld wil zijn. • In de navigatie balk zit een knop waar je kan uitloggen als een user ingelogd is.
• Als de user op de knop klikt wordt die uitgelogd		 Must
F-04 Als user wil ik een lijst van communities zien, waarin ik kan zien waar de community over gaat. • Als de user navigeert naar de communities tab zie ik een overicht van communities.
• Een my communities lijst
• Een joined communities lijst
• Een overige communities lijst
• Er moet een naam weergegeven worden
• Er moet een beschrijving weergegeven worden
• Er moet een banner foto weergegeven worden		 Must
F-05 Als user wil ik een community kunnen joinen om te zien wat voor spullen ze aanbieden • De lijst van overige communities heeft een join button. Must
F-06 Als user wil ik een community kunnen verlaten, wanneer ik geen interesse meer heb in de community. • Bij joined communities komt een knop om de community te verlaten.
• Drukken op de knop haalt je uit de community.		 Must
F-07 Als user wil ik een community kunnen aanmaken wanneer ik zie dat er nog geen community is voor mijn interesse. • In de communities tab is er een knop communities aanmaken
• De form heeft een name input
• De form heeft een description input
• De form heeft file input veld waarin je alleen png of jpg kan uploaden		 Must
F-08 Als community member wil ik de details zien van de communities waar ik deel aan neem zodat ik weet wat voor spullen ze aanbieden. • Bij de lijst van communities is er een knop om naar de detail te gaan van de community
• In de community detail pagina zie je de community name
• In de community detail pagina zie je alle members die de communities gejoined zijn.
• In de community detail pagina zie je een lijst van alle available products.		 Must
F-09 Als community owner wil de community details updaten wanneer ik vind dat het een aanpassing nodig heeft • In de community detail pagina is een update knop
• De update form is voor gevuld met de informatie van de community.
• Foto's moet je kunnen verwijderen maar ook toevoegen.		 Must
F-10 Als community owner wil ik de community deleten als er geen interesse meer in is. • In de community detail pagina is een delete knop.
• Als je op de knop klikt wordt de community gedelete		 Must
F-11 Als user wil ik een product aanmaken, zodat ik items kan uitlenen aan mensen die er interesse in heben. • In het dashboard staat een knop create item waarna je naar de add item form gaat
• In de form staat een name input
• In de form staat een bescrijving input
• In de form staat een nummer input, dit is een optionele input
• Er is een knop voor het toevoegen van images
• De images worden gedisplayed als ze geüpload zijn		 Must
F-12 Als user wil ik een product kunnen toevoegen in communities naar keuze, zodat members deze kunnen lenen wanneer ze interesse hebben. • In de product detail pagina zit er een add to community knop.
• Dit is een popup met communities die je kan aankruizen.
• De communities zijn alleen communities waaraan je deelneemt of eigenaar van bent.		 Must
F-13 Als user zou ik een product willen aanpassen, wanneer ik de informatie wil updaten. • In de product detail pagina zit er een edit product knop die je linked naar het update form.
• In de update form is alle informatie al toegevoegd
• Je moet images kunnen verwijderen en toevoegen.		 Must
F-14 Als user zou ik de details van een product willen zien, zodat ik weet wat ik zou lenen. • In een product lijst moet er een knop zijn waarmee ik kan navigeren naar een product-detail pagina.

• De product detail pagina heeft een beschrijving van het product.		 Must
F-15 Als user zou ik een product willen verwijderen wanneer ik het niet meer wil uitlenen. • De product detail pagina heeft een verwijder item knop
• Deze knop is alleen zichtbaar voor de eigenaar die ingelogd is.		 Must
F-16 Als user wil ik in mijn homepagina een overzicht hebben van communities en items zodat ik een snel startpunt heb. • Er is een homepagina voor een ingelogde user.
• In de homepagina is er een lijst van communities die gemaakt zijn door de user
• In de homepagine is een lijst van communties waar de user aan deelneemt.
• In de homepagina is een lijst van items die van de user zijn		 Could
F-17 Als user zou ik mijn product maar tot een beperkte tijd willen uitlenen, zodat het duidelijk is wanneer users het terugbrengen. • Een product heeft een max use time.
• Dit staat voor het aantal weken een user dit maximaal mag gebruiken.
• Als dit ingevuld is komt er bij elke reservering een maximale einddatum te staan		 Could
F-18 Als member van een community zou een product willen reserveren om het vervolgens op een afgesproken datum het op te halen. • Als een user in de product detail pagina is er een knop bovenaan met reserve item
• Dit is alleen als de gebruiker niet de eigenaar is.
• Dit is alleen als het item nog niet gereserveerd is.
• Als de user op de knop klikt komt er een popup.
• De popup is een form
• De popup form heeft een optioneel bericht input veld
• De popup form heeft een date of pickup datumprikker veld		 Must
F-19 Als eigenaar van een product zou ik een reservering willen accepteren of toevoegen. • Als een user een reservering doet krijgt de eigenaar een reservering te zien bij de reservations tab
• In deze reservation is er een dropdown box
• Deze dropdown box heeft een accepted en declined veld		 Should
F-20 Als user zou ik een lijst van mijn reserveringen willen zien, zodat ik weet wie mijn spullen heeft en van wie ik spullen geleend heb. • Er is een reservation tab waar de gebruiker op kan klikken
• Hierin staat een lijst met reservation aanvragen
• Hierin staat een lijst met gemaakte reservations		 Must
F-21 Als user zou ik een reservering willen cancelen als het nog niet geaccepteerd is. • In product detail staat een reservation detail
• Als het nog op pending staat kan de reservering gecancelled worden.		 Should
